# 堆
## 大顶堆和小顶堆
* 大顶堆：每个结点的值都大于或等于其左右孩子结点的值
* 小顶堆：每个结点的值都小于或等于其左右孩子结点的值

# 队列
## 优先级队列
介绍 

就是按照一定的优先规则，对队列中的元素进行排序，并返回优先级最高的元素。

头文件定义
```
#include <queue>
#include <functional> //greater<>

// 定义
priority_queue<int> pq;
```
`template <typename T, typename Container = vector<T>, typename Compare = less<T>>
class priority_queue;`
* 数据类型 T
这是队列中存放的物体。如果是基本类型（如 `int`），直接写即可；如果是自定义类，通常需要重载 `< `运算符。
* 容器类型 `Container`
虽然 `priority_queue` 表现得像队列，但它实际上是一个容器适配器。
为什么默认用 `vector`？ 因为堆排序算法（如 ``std::push_heap``）在连续内存空间上效率最高。
可选替代品：` std::deque` 也可以，但通常 `vector` 的性能更优。
* 比较准则 Compare
这是最容易让人混淆的地方。它是一个仿函数（Functor）：
`std::less<T>` (默认)：
语义：如果 a < b，则 b 的优先级更高。
结果：大顶堆（最大的元素在堆顶）。
`std::greater<T>：`
语义：如果 a > b，则 b 的优先级更高（或者说 a 被认为比 b "小"）。
结果：小顶堆（最小的元素在堆顶）。

示例
```
// 必须写 vector<int> 才能传第三个参数
priority_queue<int, vector<int>, greater<int>> minHeap;
minHeap.push(10);
minHeap.push(5);
minHeap.push(20);
cout << minHeap.top(); // 输出 5
```
重载运算符<
```
struct Node {
    int id;
    int score;
    // 重载 < 运算符
    bool operator<(const Node& other) const {
        return score < other.score; // 分数高的优先级高（大顶堆）
    }
};
priority_queue<Node> pq;
```
外部自定义比较函数
```
struct MyCompare {
    bool operator()(const Node& a, const Node& b) {
        return a.id > b.id; // id小的优先级高（类似小顶堆）
    }
};
priority_queue<Node, vector<Node>, MyCompare> pq;
```

## 使用栈的思路
栈是后进先出，对于匹配问题来说栈是优势的。最常用的一种是：**括号匹配问题**。
* **括号匹配问题**： 这是最直观的例子。在表达式 `( [ { } ] ) `中：
读取 `{`时，它是最后进入栈的。
接着读取到`}`，它立刻与栈顶的`}`匹配并消除。
这种“嵌套”关系的维持，靠的就是栈记录了“尚未匹配且离得最近”的左半部分。
* **函数调用栈**： 在编程语言执行中，函数 A 调用 B，B 调用 C。C 执行完后必须回到 B，B 执行完回到 A。这种调用与返回的匹配，本质上也是一种匹配问题。
